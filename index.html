Perguntas:

  1. Na tag < input > com atributo type="text", para que serve o atributo "placeholder"?

    2. O Placeholder resolve esse problema inserindo, dentro do input, um texto que será apagado quando começar a digitar dentro do campo, e mostrado de novo ao perder o focus do elemento e o conteúdo seja vazio.

  2. Na tag < input > com atributo type="radio" ou type="radio", para que serve o atributo "checked"?

    3. Os elementos do checked são renderizados como caixas quadradas por padrão e são marcados (com uma marca de seleção) quando ativados, assim como você vê nos formulários de governo. A aparência exata depende da configuração do sistema operacional em que o navegador está sendo executado. As caixas de seleção permitem que você selecione valores únicos para enviar no formulário

  3. Na tag < input > para que serve o atributo "value"?

    4. Você deve usar o atributo value para definir o valor enviado por este item.

  4. Que atributo deve ser utilizado para agrupar vários botões de rádio em um formulário de modo que apenas um deles fique marcado de cada vez?

    5. É utilizado o atributo < input type="radio" > que já define somente uma possibilidade de marcação.

  5. Para que serve a tag < label >? Para que serve o atributo "for" nessa tag?

    6. A tag < label > define um rótulo para vários elementos, ou seja, dá o nome dos campos a serem preenchidos no formulário. O "for" especifica a qual elemento de formulário um rótulo está vinculado.

  6. Na tag < option > para que serve o atributo "selected"?

    7. Serve para uma pré-marcação, para que ao carregar o formulário já esteja selecionado uma opção. No geral é usada na opção "Selecione" ou "Escolha" e com atributo de valor vazio, no caso de não marcação de opção do usuário.

  7. Nas tags de formulário para que serve o atributo "required"?

    8. O atributo "Required" é usado para especificar um campo de entrada que deve ser preenchido antes de enviar o formulário.
    Funciona para: texto, pesquisa,url,email,senha, data. Escrito na seguinte sintaxe:< input required>

  8. Na tag < form > para que serve o atributo "action"?

    9. O atributo "action" especifica para onde enviar os dados do formulário, seu valor deve ser um URL válido, senão serão enviados para um URL da página que contém o formulário.
    Sintaxe < form action="url" > pode ser absolute (quando coloca o endereço de um site externo) ou relative (interno)

  9. Na tag < form > para que serve o atributo "method"?

    10. É um atributo que define como os dados são enviados (os daos de formulário são enviados para a página especificada no "action").
    podem ser enviados como variáveis de URL (method="get") ou como transação de postagem HTTP (method="post").

    Observações sobre GET

    • Anexa dados do formulário ao URL em pares de nomes/valores
    • Tem comprimento limitado (3.000 caracteres)
    • Nunca use para colocar dados confidenciais
    • Útil para envios de formulários em que o usuário deseja marcar o resultado
    • Melhor usar para dados não seguros.

    Observações sobre POST

    • Anexa dados do fromulário dentro do corpo da solicitação HTTP,
    • Não tem limitação de tamanho,
    • Os envios de formulários com Post não podem ser marcados.

  10. Para que o valor digitado de uma senha não apareça na barra de endereços do navegador, qual deve ser o valor do atributo "method" na tag< form>?

    11. Para tirar proveito do “sigilo”, este tipo de campo deve ser enviado pelo método POST.

  11. Qual valor deve ser informado no atributo "method" na tag< form> em um formulário que será usado para fazer upload de arquivos?

    12. Será necessário utilizar o atributo “enctype =” multipart / form-data ” dentro da tag do formulário.
